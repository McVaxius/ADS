• No.

  TreasureFollowerDoorFollowThroughNudgeAttemptLimit only controls how many no-progress side-nudge windows before fail.

  Current code only has two actual directions:

  - attempt 1 = one side
  - attempt 2 = other side
  - then code wraps back to 1

  So:

  - Want more left/right distance: change TreasureDoorNudgeSideOffset.
  - Want more forward push: change TreasureDoorNudgeForwardOffset.
  - Want try longer before failing: change TreasureFollowerDoorFollowThroughNudgeAttemptLimit, but if set above 2, current wrap logic may keep
    repeating attempt 1/2, not create farther attempts.
